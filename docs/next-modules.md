# 다음 모듈 후보 — 사주 × Sedaily 뉴스

작성일: 2026-04-24
맥락: `/chaeun` (재운 흐름 보기) 배포 이후 다음 모듈 논의.
Sedaily 자산(MBTI 버저닝 기사 · 경제 카테고리 · 아카이브 · 일일 브리핑)을 살리고,
사주 엔진의 기존 계산 축(십성 · 대운 · 세운 · 월운 · 일진)을 재활용하는 관점으로 정리.

## 강추 3개 (우선순위 순)

### ① 직장·커리어 운 모듈 — 재운 쌍둥이

**한 줄 피치**: 재운 흐름 보기와 동일한 UX·구조로 "직장운/커리어 운"을 다룬다.

**핵심 로직 (기존 자산 재활용)**
- 엔진에 이미 **직장운 카테고리 + 관성(官星) 경로** 계산이 있음 ([categoryFortunes.ts](../frontend-next/src/features/fortune/lib/engine-data/categoryFortunes.ts), [engine-chaeun.ts](../frontend-next/src/features/fortune/lib/engine-chaeun.ts))
- `calcPeriodOverallRating`에서 관성 진입 +10점, 식상/인성 진입 가중치 이미 매핑됨
- 월운 · 세운 · 일진 3축 그대로 재활용

**Sedaily 연계**
- **경제>산업_기업** 카테고리 (DB 내 '산업' alias) → 인사 동향 · 조직 개편 · 승진 · 채용 기사
- **경제>금융_재테크** 일부 → 커리어 전환 · 연봉 · 이직 시장
- 키워드 셋: `승진 / 이직 / 채용 / 인사 / 조직 / 퇴사 / 연봉`

**매칭 규칙 (초안)**
```
관성 강 (good) → 승진·인정·조직 안정 기사
관성 중 (neutral) → 현직 유지·커리어 설계 기사
관성 약 (caution) → 리스크 관리·이직 준비·산업 구조조정 기사
```

**왜 좋은가**
- 재운보다 **덜 미신적**이고 실용적으로 느껴짐 → 사주 거부감 있는 사용자에게도 유효
- 구현 비용 **최저**: WealthNewsSection 복제 + 키워드·섹터 매핑만 교체
- Sedaily의 인사·산업 기사와 **자연스러운 정합성**

**리스크**: 거의 없음. 재운 모듈의 "사주로 돈 찍는 느낌" 피로감을 분산하는 효과도 있음.

**예상 공수**: 1~2일 (컴포넌트 구조 그대로, 매핑만 교체)

### ② 일진 기반 "오늘의 결정 가이드" + 장 마감 브리핑 — 매일 접속 훅

**한 줄 피치**: 오늘 일진 × 십성으로 "오늘 뭘 하면 좋고 뭘 피해야 하는지" + Sedaily 일일 브리핑 한 건 페어링.

**핵심 로직 (기존 자산 재활용)**
- 일진 점수 · 톤 이미 계산됨 (`periodChaeun.iljin.overall`)
- `todayFortune.ss` (오늘의 십성) · `todayFortune.us` (12운성) · `todayFortune.sinsal` (신살) 전부 엔진 내 산출
- `SS_DOMAIN_MAP` (십성 → 주 영역) 매핑 ([FortuneResult.tsx](../frontend-next/src/features/fortune/components/FortuneResult.tsx#L198)) 재활용

**UX 구조 (제안)**
```
┌──────────────────────────────────────┐
│ 📅 오늘의 결정 가이드 · 2026.04.24   │
│                                      │
│ 💪 오늘 유리한 행동                   │
│   · 계약 체결 · 프레젠테이션 · 시작    │
│                                      │
│ ⚠ 오늘 피할 행동                     │
│   · 큰 지출 · 즉흥 결정               │
│                                      │
│ ─────────────────────────────────── │
│ 🗞 오늘 Sedaily 브리핑              │
│   [장 마감 한 줄 요약 + 톱 3건]      │
└──────────────────────────────────────┘
```

**Sedaily 연계**
- `/api/briefing` (이미 있음 — [BriefingPage.tsx](../frontend-next/src/components/mbti/BriefingPage.tsx)) 일일 경제 브리핑 API 직접 사용
- MBTI 버전(`version_NT/NF/ST/SF`) 적용해 페르소나별 톤 조정

**왜 좋은가**
- 사주 사이트의 가장 큰 약점(**일회성 방문**)을 정면 돌파 — **DAU 지표** 확보
- Sedaily가 이미 매일 브리핑 생산 → **콘텐츠 비용 0**
- 푸시 알림·위젯 확장 여지 (저녁 6시 "내일의 결정 가이드" 발송 등)

**리스크**
- "오늘 유리한 행동" 카피가 너무 딱딱하거나 너무 점술적이면 둘 다 부자연스러움 → **카피 톤 조율** 중요
- 장 마감 브리핑은 평일 17:30 이후에만 의미 있음 → 주말/공휴일 fallback 콘텐츠 필요

**예상 공수**: 3~5일 (새 라우트 + 결정 가이드 카피 카탈로그 + 브리핑 API 연동 + MBTI 버저닝)

### ③ 대운(10년) × 경제 사이클 타임라인 — 콘텐츠 깊이·바이럴

**한 줄 피치**: 본인의 과거 대운 전환 시점을 실제 한국 경제 사건과 맞춰 시각화.
예: "2005년 당신의 첫 대운이 시작됐을 때, 한국 경제는 IT 버블 회복기였습니다"

**핵심 로직 (기존 자산 재활용)**
- `daeuns: DaeunEntry[]` 엔진이 이미 산출 (10년 단위)
- 각 대운 시작 연도 = 사용자별 고유 축
- 경제 사건 테이블은 정적 상수로 관리 (IMF 1997, 닷컴 2000, 글로벌 금융위기 2008, 코로나 2020 등)

**UX 구조 (제안)**
```
1995 ─────────── 2005 ─────────── 2015 ─────────── 2025 ───
       IMF(97)       글금위(08)       팬데믹(20)
         │              │                │            │
         ▼              ▼                ▼            ▼
      내 1대운      내 2대운          내 3대운     지금 (4대운)
      乙亥           丙子                丁丑          戊寅
      [학업운]       [재물운]           [관성]       [식상]
```

**Sedaily 연계**
- **타임머신 API** (`/time-machine?date=YYYY-MM-DD` 이미 존재 — [timeMachineApi.ts](../frontend-next/src/shared/api/timeMachineApi.ts)) 로 대운 시작 시점 상위 기사 1~3건 노출
- Sedaily의 **과거 기사 아카이브**가 핵심 자산 (타 서비스와 차별화 포인트)

**왜 좋은가**
- **희소성 최고** — 이 콘텐츠는 신문사 아카이브 없으면 불가능. Sedaily만의 해자
- **SNS 공유 바이럴** — "내 대운 히스토리" 스크린샷 공유 욕구 강함
- 사주 신자 아닌 사용자에게도 "내 인생과 시대의 교차점"이란 회고 콘텐츠로 어필

**리스크**
- 과거 기사 접근 범위·품질이 Sedaily 아카이브에 달려 있음 → 사전에 1997/2000/2008/2020 등 주요 연도 커버리지 확인 필요
- 대운 시작 시점이 **미래**인 경우 (20대 초반 사용자) 콘텐츠 채우기가 어려움 → 미래 대운엔 "전망" 카드로 전환

**예상 공수**: 1~2주 (경제 사건 테이블 + 타임라인 UI + 타임머신 API 연동 + 대운 × 십성 해석 카피)
